# Архитектурные решения

## 29.03.2026 — Полный рефакторинг структуры пакета `tardigradas`

### Принятые решения
- Монолитный `src/tardigradas/__init__.py` заменён модульной структурой.
- Контракт задачи вынесен в абстрактный класс `Problem`.
- Представление одной особи вынесено в класс `Individual`.
- Возврат хрупкого 6-элементного кортежа из `gen_info()` заменён на dataclass `ChromosomeSchema`.
- Генетические операторы вынесены в подпакет `operators`.
- Логика сериализации вынесена в `serialization.py`.
- Движок `Tardigradas` оставлен оркестратором и перестал быть монолитным контейнером всей логики.
- Популяция теперь хранится как `list[Individual]`, а не как `numpy`-массив Python-объектов.
- Удаление дубликатов переведено с квадратичного прохода на хеширование по `chromo.tobytes()`.

### Причины
- Улучшить поддерживаемость и навигацию по коду.
- Снизить связность между описанием задачи, представлением особи и движком.
- Упростить расширение библиотеки новыми операторами и стратегиями.
- Сделать API более явным и менее подверженным ошибкам порядка аргументов.

### Последствия
- Пользовательский API изменился и больше не совместим со старой моделью `Resolve`.
- README переписан под новый API.
- Сериализация и базовый сценарий были smoke-проверены после рефакторинга.

### Возможные следующие шаги
- Добавить автоматические тесты для `step()`, `loop()` и сериализации.
- Сделать стратегии кроссовера и мутации настраиваемыми через публичный API.
- Добавить примеры использования как отдельные файлы.
