# Анализ проекта Tardigradas

## Краткое описание
`Tardigradas` — это минималистичная Python-библиотека для решения задач оптимизации методом генетического алгоритма. Библиотека предоставляет движок эволюционного поиска и ожидает, что пользователь опишет свою предметную задачу через собственный класс-наследник `Resolve`.

Проект сейчас очень компактный и состоит фактически из одного основного модуля с ядром алгоритма, одного вспомогательного перечисления типов генов и одного файла со старой версией реализации.

## Структура проекта
- `__init__.py` — основной модуль библиотеки. Содержит:
  - перечисления `CrossoverBitType` и `CrossoverFloatType`;
  - исключение `TradigradasException`;
  - абстрактный класс `Resolve`;
  - класс `Tardigradas` с логикой популяции, селекции, кроссовера, мутаций, элитизма, сериализации и основного цикла оптимизации.
- `gen_types.py` — перечисление `GenType` (`bit`, `int`, `float`).
- `tardigradas.old.py` — устаревшая версия алгоритма, оставленная как историческая копия/референс.

## Архитектура
### 1. Класс `Resolve`
Это пользовательский слой. От него ожидается описание:
- структуры хромосомы;
- допустимых границ генов;
- необязательных значений по умолчанию;
- функции приспособленности (`fitness`);
- дополнительной валидации через `chromo_valid`.

Фактически пользователь должен реализовать:
- `gen_info(tardigradas)` — возвращает метаданные хромосомы;
- `fitness(self)` — возвращает оценку особи;
- `init_environment(tardigradas)` — вызывается из `Tardigradas.__init__`, хотя формально в `Resolve` как abstract-метод не объявлен.

### 2. Класс `Tardigradas`
Это движок генетического алгоритма. Он отвечает за:
- инициализацию популяции (`population_init`);
- оценку особей (`estimate_population`);
- ранжирование (`rank`);
- выбор родителей (`select_parents`);
- кроссовер (`crossover`, `crossover_uniform`, частично подготовлен `crossover_blx`);
- мутацию (`mutation`, `get_mutation_kid`, `mutation_gauss`);
- элитизм;
- добавление «свежей крови» (`fresh_blood_fraction`);
- удаление дубликатов (`kill_doubles`);
- сохранение и восстановление состояния (`state_dict`, `save_to_file`, `restore_from_dict`, `restore_from_file`);
- основной цикл оптимизации (`loop`).

## Модель данных хромосомы
Хромосома представляется как `numpy`-массив `float`, где реальные значения интерпретируются по типу гена:
- `bit` — бинарный ген;
- `int` — целочисленный ген;
- `float` — вещественный ген.

Дополнительно поддерживаются:
- нижние/верхние границы генов;
- группы генов (`chromo_gen_groups`) для группового переключения при uniform crossover;
- значения по умолчанию и вероятность их применения при генерации случайной особи.

## Ожидаемый публичный API
Типичный сценарий использования:
1. Создать собственный класс-наследник `Resolve`.
2. Реализовать `init_environment`, `gen_info`, `fitness`, при необходимости `chromo_valid`.
3. Создать объект `Tardigradas(resolve_class=..., population_size=..., ...)`.
4. Вызвать `population_init()`.
5. Запустить оптимизацию через `loop(...)` или пошагово через `step()`.
6. Получить лучший результат через `best_resolve`, `best_score`, `step_best_resolve`.

## Что реализовано хорошо
- Простая и компактная архитектура без лишних зависимостей кроме `numpy`.
- Поддержка смешанных типов генов (`bit`, `int`, `float`).
- Наличие элитизма, мутаций, селекции по рангу и опции «свежей крови».
- Возможность сериализовать и восстанавливать состояние поиска.
- Поддержка пользовательской валидации хромосомы через `chromo_valid()`.
- Возможность возвращать расширенный вектор оценок из `fitness`, где первая компонента используется как основная.

## Ограничения и замеченные проблемы
### API и документация
- Основной код находится прямо в `__init__.py`, что затрудняет масштабирование и навигацию.
- В проекте отсутствовали README, примеры использования, тесты и упаковочная метаинформация.
- Контракт `init_environment` используется неявно: метод вызывается, но не объявлен в базовом классе `Resolve`.

### Технические замечания
- Есть опечатка в имени исключения: `TradigradasException` вместо ожидаемого `TardigradasException`.
- В `loop()` присутствует потенциальная ошибка: при `loop_fun is None` вызывается `self.__class__.show_progress()` без аргумента `self`, что приведёт к исключению.
- Используется устаревшее `np.float` при создании массива в `crossover()`.
- Перечисления `CrossoverBitType` и `CrossoverFloatType` объявлены, но в актуальной логике почти не используются.
- Метод `estimate_population()` в текущей версии ожидает, что `fitness()` вернёт последовательность, из которой можно собрать двумерный массив; одиночное число может не соответствовать текущей реализации.

### Поддерживаемость
- `tardigradas.old.py` дублирует значительную часть логики и может создавать путаницу.
- Нет разделения на модули `core`, `operators`, `serialization`, `examples` и т. п.

## Вывод
Проект выглядит как рабочее компактное ядро генетического алгоритма для встраивания в собственные исследовательские или инженерные задачи. Его основная ценность — простота расширения через пользовательский `Resolve`. При этом библиотеке не хватает явной документации, примеров и небольшого рефакторинга API, чтобы ею было проще пользоваться как самостоятельным пакетом.

## Рекомендации по развитию
1. Вынести основной код из `__init__.py` в отдельные модули.
2. Формализовать интерфейс `Resolve`, явно добавив `init_environment` в контракт.
3. Исправить `loop()` и заменить `np.float` на встроенный `float` или `np.float64`.
4. Добавить README, примеры и минимальные тесты.
5. Решить судьбу `tardigradas.old.py`: удалить, архивировать или явно пометить как legacy.
