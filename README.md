# Tardigradas

`Tardigradas` — это компактная Python-библиотека для решения задач оптимизации с помощью генетического алгоритма.

Библиотека предоставляет ядро эволюционного поиска, а пользователь описывает свою конкретную задачу через собственный класс-наследник `Resolve`: определяет структуру хромосомы, ограничения, функцию приспособленности и, при необходимости, правила валидации.

## Что есть в проекте

- поддержка трёх типов генов: `bit`, `int`, `float`;
- инициализация популяции случайными особями;
- элитизм;
- ранговый отбор родителей;
- кроссовер;
- мутация;
- добавление «свежей крови» в популяцию;
- удаление дубликатов;
- сохранение и восстановление состояния поиска через `pickle`.

## Идея использования

Пользователь реализует свой класс задачи на основе `Resolve`, после чего передаёт его в `Tardigradas`.

### Что нужно реализовать в своём классе

Минимально ожидается следующее:

- `init_environment(tardigradas)` — инициализация окружения задачи;
- `gen_info(tardigradas)` — описание хромосомы;
- `fitness(self)` — функция приспособленности;
- `chromo_valid(self)` — опционально, если нужны дополнительные ограничения.

> Обратите внимание: `init_environment` в текущем коде вызывается движком, но формально не объявлен как `abstractmethod` в базовом классе `Resolve`.

## Формат `gen_info`

Метод `gen_info` должен вернуть:

1. список типов генов;
2. кортеж из двух списков: минимальные и максимальные границы;
3. комментарии к генам;
4. группы генов;
5. значения по умолчанию;
6. вероятности применения значений по умолчанию.

Пример возвращаемой структуры:

```python
return (
    [GenType.float, GenType.float, GenType.int],
    ([0.0, 0.0, 0], [10.0, 10.0, 5]),
    ["x", "y", "mode"],
    [0, 0, 1],
    [1.0, float("nan"), 2],
    [0.3, 0.0, 0.5],
)
```

## Важная деталь про `fitness`

В текущей реализации `fitness()` должен возвращать **последовательность чисел** (например, список, кортеж или `numpy`-массив), где:

- первый элемент — основная оценка, по которой идёт оптимизация;
- остальные элементы — дополнительные метрики, если они нужны.

Пример:

```python
return [score]
```

или

```python
return [score, penalty, aux_metric]
```

## Пример использования

```python
from tardigradas import Resolve, Tardigradas
from tardigradas.gen_types import GenType


class SphereResolve(Resolve):
    @staticmethod
    def init_environment(tardigradas):
        # Можно подготовить дополнительные данные окружения.
        # В этом примере ничего не нужно.
        pass

    @staticmethod
    def gen_info(tardigradas):
        return (
            [GenType.float, GenType.float],
            ([-5.0, -5.0], [5.0, 5.0]),
            ["x", "y"],
            [0, 0],
            [float("nan"), float("nan")],
            [0.0, 0.0],
        )

    def fitness(self):
        x = self[0]
        y = self[1]

        # Алгоритм максимизирует score,
        # поэтому для минимизации x^2 + y^2 берём отрицательное значение.
        score = -(x ** 2 + y ** 2)
        return [score]

    def chromo_valid(self):
        return True


ga = Tardigradas(
    resolve_class=SphereResolve,
    population_size=50,
    crossover_fraction=0.5,
    fresh_blood_fraction=0.1,
    gen_mutation_fraction=0.1,
    fitness_environment=None,
    n_elits=2,
)

ga.population_init()

# Важно: в текущей версии лучше явно передать loop_fun,
# так как поведение по умолчанию в loop() реализовано неудачно.
ga.loop(max_iterations=100, loop_fun=lambda ga: False)

print("Лучшая оценка:", ga.best_score)
print("Лучшее решение:", ga.best_resolve.chromo)
```

## Основные параметры `Tardigradas`

- `resolve_class` — пользовательский класс задачи;
- `population_size` — размер популяции;
- `crossover_fraction` — доля новых особей, рождающихся через кроссовер;
- `fresh_blood_fraction` — доля новых случайных особей в поколении;
- `gen_mutation_fraction` — интенсивность мутации по числу изменяемых генов;
- `fitness_environment` — произвольное окружение для функции приспособленности;
- `n_elits` — число элитных особей, переходящих в следующее поколение без изменений.

## Полезные методы

### Инициализация и шаги алгоритма

- `population_init()` — создать начальную популяцию;
- `step()` — выполнить одну итерацию;
- `loop(...)` — запустить цикл оптимизации.

### Работа с результатами

После запуска оптимизации доступны:

- `best_score` — лучшая найденная оценка;
- `best_iteration` — итерация, на которой найден лучший результат;
- `best_resolve` — лучшая найденная особь;
- `step_best_resolve` — лучшая особь на текущем шаге.

### Сохранение состояния

```python
ga.save_to_file("state.pkl")
```

### Восстановление состояния

```python
ga.restore_from_file("state.pkl")
```

## Ограничения и особенности текущей версии

- основная логика находится прямо в `__init__.py`;
- в проекте нет тестов и примеров как отдельных файлов;
- `tardigradas.old.py` содержит старую версию реализации и может путать;
- в коде есть потенциальные шероховатости:
  - опечатка в имени исключения `TradigradasException`;
  - использование устаревшего `np.float`;
  - неявный контракт метода `init_environment`;
  - возможная ошибка в поведении `loop()` по умолчанию.

## Для чего проект подходит

Проект хорошо подходит как:

- лёгкое ядро генетического алгоритма для собственных экспериментов;
- база для учебных задач по эволюционным методам;
- отправная точка для дальнейшего расширения под конкретную прикладную область.

Если планируется использовать библиотеку как полноценный пакет, стоит добавить тесты, примеры, более явный API и разнести код по отдельным модулям.
